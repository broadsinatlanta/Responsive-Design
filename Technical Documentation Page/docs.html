Python Data Types Documentation

Introduction

Basic Python Data Types:

  • Integer (int)
  • Float
  • String (str)
  • List
  • Dictionary (dict)
  • Booleans (bool)
  • NoneType

Python stores data in assigned values named variables. Each variable represents a part of data that, at its core, behaves and performs in line with one of the above data types.

Each data type can be expected to adhere to its own rules and use cases, and are powerful tools to carry out logic in your code.

Using Different Data Types In Your Code

Though not exhaustive, the list above covers the vast majority of data types you will need to use at a medium to low level. Python offers other data types, as well as the ability to create your own, but the concepts needed to understand them are gained through familiarity of the ones above.

Assign A Variable To Data

message = "Hello World!"

print(message)
Hello World! # prints out in console

Quick Run through

Int and float data types are used to represent numbers and can be used to carry out most mathematical functions. Int represents a positive or negative whole number while a float represents a positive or negative number with a decimal point (note. A whole number with a x.0 decimal is still a float).

A string represents a string of characters and are used to represent written data.

Lists and dicts are used to store multiple variables or unassigned data in one reference variable. Lists and dicts in essence exist to the same point but are referenced and indexed in different ways. Lists are ordered, while dicts use a key-value system.

Boolean values exist as either true or false. Booleans are used, among others, to check conditions and assess process outcomes.

NoneType values are used to indicate null values: values that exist but have no value (not a value of 0). NoneType values have a value of None.

Checking The Type Of A Value

print(type(message))
<class 'str'> # prints out in console

Numbers

Two Types

As stated in the introduction, int variables are whole numbers and float variables are numbers with decimal points. Both float and int can be either positive or negative.

Numbers In Python

x = 2.5
y = 5

print(type(x))
<class 'float'> # prints out in console

print(type(y))
<class 'int'> # prints out in console

Numeric Operations

Int and float values can be used to perform a wide range of mathematical functions, he most basic of which can be carried out using operators.

Int variables will remain int as long as the other(s) variables in the operation are also int. The only time this doesn't hold true is if the answer is, by nature, a float. Otherwise, if any of the variables are floats, the end result will also be a float.

Operating On Float And Ints

a = x + x

print(a, type(a))
5.0 <class 'float'> # prints out in console

b = y + y
print(b, type(b))
10 <class 'int'> # prints out in console

c = x + y
print(c, type(c))
7.5 <class 'float'> # prints out in console

Other mathematical Operations:

  • Subtraction: (x - y)
  • Multiplication: (x * y)
  • Division: (x / y)
  • Modulus: (x % y)
  • Exponent: (x ** y)
  • Floor Division*: (x // y)

All the above operations follow the same type interaction rules.

* - Normal division but all decimal values are dropped before returning.

Strings

Strings vs Characters

A character is used to represent any symbol as it looks in your process. As a consequence, a character can be any symbol - including numbers, spaces and operators.

A string is a collection of character values pasted into one longer string. There is no technical distinction between the two in Python: a character is a string that is one character long.

Message

message = "Hello World 2!!"

print(message, type(message))
Hello World 2!! <class 'str'> # prints out in console

Operators On Strings

Strings can be added together and multiplied with values using operators.

Operating On Strings

a = "Python"
b = " is easy."
c = a + b

print(c)
Python is easy. # prints out in console

d = a * 3

print(d)
PythonPythonPython # prints out in console

Usage

Strings are often used as user input or to denote qualitative values.

Lists

Grouping Data

Lists are used to store groups of data in one variable. Lists, however, aren't limited by data type; they can store any and all data types within one instance.

Lists can, in any one instance, carry as many items in it as you like and have no limit on the number of different data types it can hold. Lists can also be empty.

Initialising Lists

emptyList = []
fullList = ["a", 2, "3+"]

print(type(emptyList), type(fullList))
<class 'list'> <class 'list'>

Indexing Lists

Lists are ordered and therefore can be numerically indexed. Every Every item in the list is assigned an ascending number, starting at 0.

You can then use the index to retrieve the value at that index using [].

Retrieving Listed Items

print(fullList[0])
a # prints out in console

print(fullList[1])
2 # prints out in console

print(fullList[2])
3+ # prints out in console

Subsetting Lists

[] syntax can be used to retrieve multiple items in a list, returning desired items in list form. Lists can be indexed using integers or slices. A slice is used using x:y syntax - where xth is included and yth value is excluded.

Subsetting Lists

print(fullList[0:2])
["a", 2] # prints out in console

Negative integers can be used in slices and as indices and work in the same way. Using negative indices index the list from the end.

Negative indices

print(fullList[-3:-2])
["a", 2] # prints out in console

print(fullList[-1])
["3+"] # prints out in console

Using indexing, you can also assign values at an index using assignment.

List Assignment

fullList[0] = "change"

print(fullList[0])
change # prints out in console

Unique Properties

As its own data type or class, lists have properties and methods. These are built in attributes and capabilities that can be utilised by you, by virtue of being a list.

Methods & Attributes

# Get the index of an item
print(fullList.index("change"))
0 # prints out in console

# Add item onto a list
fullList.append("new")
print(fullList) ["change", 2, "3+", "new"] # prints out in console

There are many more methods and attributes available to lists which can be found here.

Strings As Lists?

Strings are very similar to lists: strings are essentially lists of characters. They can therefore be indexed like lists and share some methods and attributes.

Strings do have their differences, mostly in how you mutate them. Lists don't respond to the + or * operators and cannot be mutated at an index. Strings are also hashable where lists aren't.

Strings vs Lists

message = "String"
message2 = ["S", "t", "r", "i", "n", "g"]

message2[0] = "s" # message2 is ["s", "t", "r", "i", "n", "g"]
message[0] = "s" # Returns an error

Read more about their differences here.

Dictionaries

Similar To Lists

Dictionaries are a data type intended to hold groups of data. Unlike lists, dictionaries (dicts) are unordered and are indexed using a key - value system. Every time an item is added to a dict, it is assigned a key - which can be any data object that is hashable (mainly strings or integers).

As a consequence, only use dicts when the order doesn't matter to the process and you need to establish a specific relationship between the data and a key - e.g. an id number with a name.

Making A Dict

emptyDict = {}
liverpool = {"Salah" : 11, "van Dijk" : 4, "Alisson" : 13}

print(type(liverpool))
<class 'dict'> # prints out to console

Booleans

True Or False

Booleans are the virtual representation of the concept of true and false values.

The 2 Booleans

yes = True
no = False

print(yes, no)
True False # prints out to console

print(type(yes), type(no))
<class 'bool'> <class 'bool'> # prints out in console

A true value would be anything of substance i.e. it is a real value with real use. A false value is the opposite - something of no actual value or an empty data type.

Mostly every value is considered a true value except:

  • None
  • False
  • Numeric zeros
  • Empty data structures i.e. lists or dicts

Booleans mainly find use in conditionals.

NoneTypes

NoneType is the class of data which symbolises the opposite of value - not 0, which is a value of nothing, but no value whatsoever.

An NoneType object has a value of None and is rarely used by the user.
